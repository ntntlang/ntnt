The NTNT Programming Language

A programming language for building full-stack systems with AI. NTNT includes a comprehensive standard library and native Intent-Driven Development (IDD), enabling AI agents to validate code against explicit requirements.

Why NTNT?

A modern programming language designed for AI-assisted development with built-in verification.

Full-Featured Language

Not a toy. NTNT has contracts, pattern matching, generics, enums, structs, traits, and a comprehensive standard library. Build real applications with HTTP servers, PostgreSQL, JSON, CSV, file I/O, crypto, and concurrency.

Intent-Driven Development

Write requirements in .intent files. Link code with @implements. Run ntnt intent check to verify they match. Full traceability from requirement to implementation.

Design by Contract

requires and ensures are built into the syntax. Agents read them as specs. Humans read them as docs. In HTTP routes, contract violations automatically return proper error responses.

Intent Studio

Visual development environment with live test execution. See pass/fail indicators update in real-time as you code. One command: ntnt intent studio server.intent

Native Hot-Reload

HTTP servers automatically reload when you save. No restart needed. Edit your code, refresh the browser, see changes instantly.

Agent-Native Tooling

ntnt inspect outputs structured JSON describing every function, route, and contract. AI agents understand your codebase in a single call.

What It Looks Like

A complete web API in NTNT:

// api.tnt
import \{ json \} from "std/http/server"
import \{ connect, query \} from "std/db/postgres"

let db = connect("postgres://localhost/myapp")

fn get_user(id: Int) -> Map
    requires id > 0
    ensures result["id"] == id
\{
    return query(db, "SELECT * FROM users WHERE id = $1", [id])[0]
\}

// @implements: feature.user_api
get(r"/users/\{id\}", get_user_handler)

listen(3000)

Standard Library

Batteries included. No package manager needed.

Web

HTTP server with routing, middleware, static files. HTTP client with fetch, auth, and cookies.

Database

Native PostgreSQL with parameterized queries, transactions, and connection pooling.

Data Formats

JSON and CSV parsing and serialization. URL encoding and query string handling.

File System

Read, write, copy, move files. Directory operations. Path manipulation.

Crypto

SHA256, HMAC, UUID generation, random bytes, hex encoding.

Concurrency

Go-style channels with send, recv, and timeouts. No async/await complexity.
