About NTNT

Reimagining software development for an era where AI agents handle the heavy lifting of coding, testing, and deployment.

Why NTNT?

Traditional programming languages were designed for human developers. They include complexities that matter to humans but create friction when AI agents write code. NTNT bridges this gap with a focus on formal verification and machine-readability.

Unambiguous Syntax

Deterministic parsing for reliable AI code generation and manipulation without the ambiguity of traditional syntax.

Formal Contracts

Explicit preconditions and postconditions as first-class language features, ensuring safety by design.

Structured Edits

Powerful AST-based code manipulation allows AI agents to perform safe refactoring and transformations.

Human Oversight

Built-in approval gates and transparent execution logs ensure critical decisions always involve human developers.

Core Verification Features

Function Contracts

Every function can specify what it requires and what it ensures, allowing the runtime to catch logic errors immediately before they propagate.

fn divide(a: Int, b: Int) -> Int
    requires b != 0  // Precondition
    ensures result * b == a  // Postcondition
{
    return a / b
}

Struct Invariants

Structs can declare global invariants that are checked automatically on every mutation, preventing invalid state from ever being persisted.

struct BankAccount {
    balance: Int,
    owner: String
}

impl BankAccount {
    invariant self.balance >= 0
}

HTTP Server

Build web APIs with contract-verified endpoints. Violations automatically return appropriate HTTP error codes.

fn create_user(req)
    requires req.body != ""  // Returns 400 if violated
{
    return json(map { "created": true })
}

get("/users", list_users)
post("/users", create_user)
listen(3000)
