About NTNT

NTNT (/ɪnˈtɛnt/) reimagines software development for an era where AI agents handle the heavy lifting of coding, testing, and deployment.

Why NTNT?

Traditional programming languages were designed for human developers. They include complexities that matter to humans but create friction when AI agents write code. NTNT bridges this gap with:

  • Unambiguous syntax - Deterministic parsing for reliable AI code generation
  • Formal contracts - Explicit preconditions and postconditions as first-class features
  • Structured edits - AST-based code manipulation for safe refactoring
  • Human oversight - Built-in approval gates for critical decisions

Core Features

Contracts

Every function can specify what it requires and what it ensures:

fn divide(a: Int, b: Int) -> Int
    requires b != 0  // Precondition
    ensures result * b == a  // Postcondition
{
    return a / b
}

Struct Invariants

Structs can declare invariants that are checked automatically:

struct BankAccount {
    balance: Int,
    owner: String
}

impl BankAccount {
    invariant self.balance >= 0
}

HTTP Server

Build web APIs with contract-verified endpoints:

fn create_user(req)
    requires req.body != ""  // Returns 400 if violated
{
    return json(map { "created": true })
}

get("/users", list_users)
post("/users", create_user)
listen(3000)
